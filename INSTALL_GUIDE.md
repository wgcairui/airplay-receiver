# 📱 PadCast 安装指南

本指南将帮助您在 OPPO Pad 或其他 Android 平板设备上安装和配置 PadCast 应用。

## 📋 系统要求

### 🎯 推荐设备
- **OPPO Pad 4 Pro (13.2英寸)** - 原生优化支持
- **OPPO Pad Air** - 完全兼容
- **小米平板 6 Pro** - 良好兼容性
- **华为 MatePad Pro** - 基础兼容

### 🔧 最低要求
- **操作系统**: Android 8.0+ (API level 26+)
- **内存**: 至少 2GB RAM (推荐 4GB+)
- **存储空间**: 至少 100MB 可用空间
- **网络**: WiFi 连接必需
- **处理器**: ARM64 架构 (推荐)

### 📶 网络要求
- **WiFi**: 2.4GHz 或 5GHz (推荐 5GHz)
- **带宽**: 至少 20Mbps (4K 内容需要 50Mbps+)
- **延迟**: 低于 50ms 到路由器
- **同网络**: 投屏设备需在同一局域网

## 🚀 安装方法

### 方法一: 直接下载 APK（推荐）

#### 步骤 1: 下载 APK 文件

1. **访问发布页面**
   - 打开浏览器，访问: [GitHub Releases](https://github.com/cairui/padcast/releases)

2. **选择版本**
   - 选择最新的稳定版本 (标记为 "Latest")
   - 避免选择标记为 "Pre-release" 的测试版本

3. **下载 APK**
   - 点击 `padcast-v1.0.0-arm64-v8a.apk` (推荐，适用于大多数现代设备)
   - 或下载 `padcast-v1.0.0-armeabi-v7a.apk` (适用于较旧设备)
   - 下载大小约 50MB

#### 步骤 2: 启用未知来源安装

1. **打开设置**
   - 在 OPPO Pad 上打开 "设置"
   - 导航到 "安全" 或 "隐私"

2. **允许未知来源**
   - 找到 "未知来源" 或 "安装未知应用"
   - 允许从浏览器或文件管理器安装应用
   - 部分设备需要为特定应用单独授权

#### 步骤 3: 安装 APK

1. **定位文件**
   - 打开 "文件管理器" 或 "下载"
   - 找到下载的 `padcast-v1.0.0-arm64-v8a.apk` 文件

2. **开始安装**
   - 点击 APK 文件
   - 系统会显示安装确认对话框
   - 确认应用权限和信息

3. **完成安装**
   - 点击 "安装" 按钮
   - 等待安装完成（通常需要 10-30 秒）
   - 看到 "应用已安装" 提示

## ⚙️ 初始配置

### 首次运行设置

1. **启动应用**
   - 点击 "打开" 或在应用抽屉中找到 PadCast
   - 应用图标为蓝色的投屏标志

2. **授权权限**
   - 网络访问权限（必需）
   - 存储访问权限（用于日志和设置）
   - 通知权限（用于状态提醒）

3. **网络检测**
   - 应用会自动检测网络连接
   - 确保设备已连接到 WiFi 网络
   - 检查网络质量和延迟

### 推荐配置

#### 🎥 视频设置
```
质量模式: 高质量 (1080p@60fps)
编解码器: H.264 (兼容性最好)
硬件加速: 启用 (推荐)
```

#### 🎵 音频设置
```
质量模式: 高质量 (320kbps)
音频增强: 启用
同步阈值: 50ms
```

#### 🌐 网络设置
```
连接模式: 自动检测
低延迟模式: 启用
自动重连: 启用
连接超时: 15秒
```

## 🔧 网络配置

### 路由器设置

#### 必需配置
1. **启用 mDNS/Bonjour**
   - 确保路由器支持 mDNS 服务发现
   - 某些企业路由器可能默认禁用

2. **设备间通信**
   - 允许同一网络内设备相互通信
   - 禁用 AP 隔离或客户端隔离

3. **端口开放**
   - TCP 端口: 7000-7010 (AirPlay 服务)
   - UDP 端口: 5353 (mDNS)
   - TCP 端口: 80, 443 (Web 服务)

#### 性能优化
1. **使用 5GHz 频段**
   - 更少干扰，更高带宽
   - 更好的投屏体验

2. **QoS 设置**
   - 为 AirPlay 流量设置高优先级
   - 限制其他设备的带宽使用

## 📱 设备特殊设置

### OPPO Pad 专用设置

#### 系统优化
1. **性能模式**
   - 设置 > 电池 > 性能模式 > 高性能模式
   - 确保 CPU 和 GPU 在最佳状态

2. **网络优化**
   - 设置 > WiFi > 高级设置 > WiFi 休眠策略 > 从不
   - 避免投屏时网络断开

3. **通知设置**
   - 允许 PadCast 发送通知
   - 设置为重要通知级别

#### ColorOS 特殊设置
1. **后台保护**
   - 设置 > 应用管理 > PadCast > 权限管理
   - 允许后台运行和自启动

2. **省电白名单**
   - 设置 > 电池 > 省电模式 > 应用省电管理
   - 将 PadCast 加入白名单

## 🔍 安装验证

### 功能测试

#### 基础功能验证
1. **启动测试**
   - 打开 PadCast 应用
   - 确认界面正常显示
   - 检查服务状态为 "等待连接"

2. **网络测试**
   - 查看网络状态指示器
   - 确认显示正确的 IP 地址
   - 网络类型应为 WiFi

3. **设备发现测试**
   - 在发送端设备上打开 AirPlay
   - 确认能看到 "PadCast" 设备
   - 尝试建立连接

#### 性能测试
1. **自动化测试**
   - 打开设置 > 自动化测试
   - 运行 "基础功能测试"
   - 确保所有测试通过

2. **手动投屏测试**
   - 从 iPhone/Mac 投屏测试视频
   - 检查画质和流畅度
   - 确认音视频同步

### 故障排除

#### 安装失败
- **"应用未安装"**: 启用未知来源安装
- **"解析包时出现问题"**: 重新下载 APK 文件
- **存储空间不足**: 清理设备存储空间

#### 权限问题
- **网络权限被拒绝**: 手动在应用设置中授权
- **无法写入存储**: 检查存储权限设置
- **通知不显示**: 在通知设置中允许通知

#### 兼容性问题
- **设备不支持**: 确认 Android 版本 ≥ 8.0
- **架构不匹配**: 下载对应架构的 APK
- **性能不足**: 降低视频质量设置

## 🔄 更新指南

### 自动更新检查
PadCast 会定期检查更新：
- 启动时检查新版本

- 有新版本时会显示通知

1. 访问 [GitHub Releases](https://github.com/cairui/padcast/releases)
2. 下载最新版本 APK
3. 直接安装覆盖旧版本
4. 设置和数据会自动保留

- 向后兼容: 新版本兼容旧版本设置
- 数据迁移: 自动迁移用户配置
